## ADDED Requirements

### Requirement: 学习记录页面路由注册

系统 SHALL 在 `pages.json` 中注册 `pages/study-record/index` 页面路由，导航栏标题为"学习记录"。

#### Scenario: 从个人中心进入学习记录
- **WHEN** 用户在个人中心点击"学习记录"菜单项
- **THEN** 导航到学习记录页面，显示导航栏标题"学习记录"

### Requirement: 统计摘要卡片

页面顶部 SHALL 展示统计摘要卡片，包含本月学习时长、本月预约次数、最长连续学习天数、累计学习时长四项数据。卡片样式 MUST 参照 `prototype/study-record.html` 中的渐变背景卡片设计。

#### Scenario: 有学习数据时展示统计
- **WHEN** 页面加载且用户有已完成的学习记录
- **THEN** 顶部渐变卡片显示四项统计数据，数值从 summary API 获取

#### Scenario: 无学习数据时展示
- **WHEN** 页面加载但用户无学习记录
- **THEN** 统计卡片四项数值均显示 0

### Requirement: 月度学习日历

页面 SHALL 展示月度日历视图，标记已学习日期和今日。用户可切换月份查看历史学习记录。

#### Scenario: 查看当月日历
- **WHEN** 页面首次加载
- **THEN** 显示当前月份日历，已学习的日期以绿色背景标记，今日以主色蓝圆圈标记，未来日期灰色显示

#### Scenario: 切换月份
- **WHEN** 用户点击月份左右箭头
- **THEN** 日历切换到对应月份，summary API 重新请求该月数据，日历标记更新

### Requirement: 学习记录列表

页面 SHALL 在日历下方展示最近学习记录列表，每条记录显示门店名称、座位号、时间段、学习时长、消费金额。

#### Scenario: 查看记录列表
- **WHEN** 页面加载
- **THEN** 按日期倒序显示学习记录，每条记录包含门店名称、座位编号、时间段、时长、金额

#### Scenario: 加载更多记录
- **WHEN** 用户滚动到列表底部
- **THEN** 自动加载下一页记录，直到无更多数据

### Requirement: 页面加载状态

页面 MUST 在数据加载过程中显示加载状态提示。

#### Scenario: 首次加载
- **WHEN** 页面首次打开，数据尚未返回
- **THEN** 显示加载中提示
